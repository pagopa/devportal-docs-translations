# Remote configuration

This configuration is required if you plan to use one of the following features:

1. [send-a-message-with-remote-content.md](../features/send-a-message/send-a-message-with-remote-content.md "mention")
2. [add-attachments.md](../features/send-a-message/add-attachments.md "mention")

For both features, you must **make available a **_**&#x52;EST web service**_ that complies with the [relevant OpenAPI](../api-and-specifications/openapi-remote-content-retrieval-endpoint.md).

An exposed web service is characterized by a **configuration identifier**, which is a unique alphanumeric code generated by IO that will correspond to a set of endpoints and their related API keys.

{% hint style="info" %}
You must use the specific [`manage`](../Features/Publishing-a-service/Manage-key/Manage-key.md) key to manage configurations: find out [how to retrieve it](../Features/Publishing-a-service/Manage-key/Manage-key.md#recupera-la-chiave-manage)
{% endhint %}

You can define a configuration, recall it later, list the defined configurations, and modify one that was previously created.

{% hint style="info" %}
You can also define **more than one configuration** within the scope of the [manage key](../Features/Publishing-a-service/Manage-key/Manage-key.md) in use.

For example, if your Organization uses multiple Technology Partners, each exposing a REST web service to serve remote features, a different configuration identifier can be defined for each to be used later to [send messages](../APIs-and-specifications/Message-APIs/Submit-a-message-passing-the-user-fiscal-code-in-the-request-body.md) under their own responsibility.
{% endhint %}

{% hint style="success" %}
To manage configurations, you must use the API described in [cru-configurazioni-remote.md](../api-e-specifiche/api-messaggi/cru-configurazioni-remote.md "mention").
{% endhint %}

{% hint style="success" %}
**Each remote content message you send will refer to a specific configuration**: to indicate it, you must use the mandatory [#configuration\_id](../APIs-and-specifications/Message-APIs/Submit-a-message-passing-the-user-fiscal-code-in-the-request-body.md#configuration_id "mention") parameter during creation.
{% endhint %}

The exposed service must consist of a variable number of _endpoints_, depending on what you intend to remotize for your messages and the [type of agreement](https://docs.pagopa.it/kb-enti-accordi/domande-frequenti/domande-e-risposte-sugli-accordi#quali-sono-le-formule-contrattuali-possibili-per-aderire-allapp-io) you have signed with PagoPA:

<table><thead><tr><th>Retrieval endpoint for:</th><th width="275">Data to be returned to IO</th><th>Details</th></tr></thead><tbody><tr><td>opening preconditions</td><td>title and body of the preconditions</td><td></td></tr><tr><td>message details</td><td><ul><li>title and body of the message;</li><li>attachment metadata (Premium entities only)</li></ul></td><td>the message details must match the intent flags declared when creating the message</td></tr><tr><td>attachment bytes</td><td>bytes of the single attachment (Premium entities only)</td><td></td></tr></tbody></table>

{% hint style="info" %}
For more information on how to design the _endpoints_ and the related exposed API, refer to [openapi-endpoint-di-recupero-dei-contenuti-remotizzati.md](../api-e-specifiche/openapi-endpoint-di-recupero-dei-contenuti-remotizzati.md "mention").
{% endhint %}

Once ready, use the API described in [cru-configurazioni-remote.md](../api-e-specifiche/api-messaggi/cru-configurazioni-remote.md "mention") to register the necessary data with the IO systems, including:

- the **base URL** that IO will use to compose the addresses of the _endpoints_ exposed by your Organization (for more information, refer to the examples in [openapi-endpoint-di-recupero-dei-contenuti-remotizzati.md](../api-e-specifiche/openapi-endpoint-di-recupero-dei-contenuti-remotizzati.md "mention"))
- The **API key** that IO will use to authenticate with your systems and its corresponding expected header
- any default preference regarding message opening preconditions (for more information on preconditions, refer to [inviare-un-messaggio-a-contenuto-remoto.md](../funzionalita/inviare-un-messaggio/inviare-un-messaggio-a-contenuto-remoto.md "mention"))

{% hint style="warning" %}
To ensure the long-term consistency and proper functioning of remote content messages in the app, **it is not possible to delete a configuration** once it has been defined.

Furthermore, for the same reason, it is the sender's responsibility to **promptly update the configurations** whenever necessary, using the management API described in [cru-configurazioni-remote.md](../api-e-specifiche/api-messaggi/cru-configurazioni-remote.md "mention").
{% endhint %}
