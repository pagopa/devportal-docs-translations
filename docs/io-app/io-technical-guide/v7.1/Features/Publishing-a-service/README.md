# 📦 Publishing a service

A service on IO is the **digital version**, or an **extension of an already digitized service**, provided by an entity – public or private, central or local – and intended for one or more groups of citizens.

Here are the steps to follow to create a service.

<details>

<summary><mark style="color:blue;">Step 1</mark> - Find out what a service is</summary>

Consult the [IO app Services Manual](https://docs.pagopa.it/manuale-servizi/) to find out:

- **what is meant by service** in the context of IO;
- what **types of services** can be provided in IO and **what features** can be used;
- get instructions on **how to create** services;
- consult **service templates** you can use.

</details>

<details>

<summary><mark style="color:blue;">Step 2</mark> - Create the service</summary>

Follow the instructions on how to "[Create a new service](Creating-a-service.md)"

</details>

<details>

<summary><mark style="color:blue;">Step 3</mark> - Test the service</summary>

Before publishing the service in the app, view the service details in the app to make sure everything is correct. To do so, read [provare-un-servizio-in-test.md](provare-un-servizio-in-test.md "mention")

</details>

<details>

<summary><mark style="color:blue;">Step 4</mark> - Send a test message</summary>

If you want, you can also send a test message. To learn more about messages and how to send test messages, read how to [inviare-un-messaggio](../inviare-un-messaggio/ "mention") and send [messaggi-di-test.md](../inviare-un-messaggio/messaggi-di-test.md "mention")

</details>

<details>

<summary><mark style="color:blue;">Step 5</mark> - Request the service review</summary>

If everything is ready, [submit the service for review](Service-review.md) by PagoPA. If the service is evaluated positively it will be **approved**; otherwise it will be **rejected** with a reason, which will be communicated to you so that you can modify it and resubmit it.

</details>

<details>

<summary><mark style="color:blue;">Step 6</mark> - Publish the service</summary>

If you did not request automatic publication during the review, once approved you can [make the service visible](Service-visibility.md) yourself.

</details>
