# Introduction

This tutorial guides you through the **configuration of the "Groups" feature in the Reserved Area**, which allows you to define the visibility of IO services for one or more users (operators).

### What is a group? <a href="#what-is-a-group" id="what-is-a-group"></a>

A group is a **set of users from an organization who are grouped according to a common characteristic.** For example, a group of users can represent a specific office or a technology partner that supports the organization.

See the [Reserved Area Operational Manual](https://docs.pagopa.it/area-riservata/area-riservata/come-funziona/gruppi) for more details on Groups.

See the [Technical Guide](https://app.gitbook.com/o/KXYtsf32WSKm6ga638R3/s/AIHi4SHY3sa8Or3prCOe/funzionalita/pubblicare-un-servizio/gestire-laccesso-ai-servizi-tramite-i-gruppi) for more details on the feature in relation to IO.&#x20;

### What are the consequences of creating a group for IO? <a href="#what-are-the-consequences-of-creating-a-group-for-io" id="what-are-the-consequences-of-creating-a-group-for-io"></a>

As an **administrator**, you can use groups to limit the scope of visibility and operation for operator users on certain IO services. The administrator will always have full visibility, regardless of whether they belong to one or more groups.

As an **operator**, you can only manage and see the services that have been assigned to the groups you are part of. If an operator is not associated with any group, they will see all the organization's services without restriction.
