# Client management

The client system is highly flexible to allow users to organize clients according to their own processes, rather than adapting processes to clients. Different implementation approaches are possible:

* one client per e-service;
* multiple clients per e-service;
* a single client for multiple e-services.

Each entity will make a trade-off between security and maintainability of its own solution. In principle, the organization of clients associated with e-services must ensure that technicians have visibility over the purposes for which they operate, but not others.

For more information, please see the [webinar dedicated to clients](https://developer.pagopa.it/webinars/e-service-erogazione-inversa) (from minute 24:20 — free subscription, Italian only).

{% hint style="warning" %}
It is recommended to assign names to clients that help identify the group of people using them (for example, employees of an external or internal software house to the party).
{% endhint %}
