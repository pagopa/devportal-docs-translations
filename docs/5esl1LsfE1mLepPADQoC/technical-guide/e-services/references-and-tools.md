# References and tools

{% hint style="info" %}
Please note that all the following links are Italian only.
{% endhint %}

## Guide to naming e-services

The Department for Digital Transformation has created a dedicated guide for e-service nomenclature, consisting of a set of best practices for naming and describing them on PDND.

The goal is to make services more easily searchable and identifiable.

Members are required to follow the guidelines provided in the guide, available at the following link: [https://italia.github.io/pdnd-guida-nomenclatura-eservice/](https://italia.github.io/pdnd-guida-nomenclatura-eservice/).

## API validator

The Department for Digital Transformation has published a tool that allows verification of the compatibility of one’s REST API interface with the requirements set out in AgID’s ModI.

Members are required to verify that their interface file passes the validator’s checks before publishing their e-service.

The API validator is available at the following link: [https://italia.github.io/api-oas-checker/](https://italia.github.io/api-oas-checker/), and is accompanied by [Usage guides](https://github.com/italia/api-oas-checker-rules).

## Semantic validator

The Department for Digital Transformation, together with Istat, has published a tool to verify the metadata of semantic resources present in interface files.

The validator is part of a broader project:

> Schema is the continuously evolving catalog that harmonizes and standardizes shared data models, ensuring that the format and meaning of the exchanged information are preserved and understood during exchanges.
>
> — [schema.gov.it](https://schema.gov.it/)

The semantic validator is available at the following link: [https://schema.gov.it/validatore\_metadati](https://schema.gov.it/validatore_metadati).

## Developing APIs compliant with ModI

For some articles on API development, refer to the dedicated [news section](https://developers.italia.it/it/news/interoperabilita.html) on Developers Italia. Also, see the [ModI page](https://www.agid.gov.it/it/infrastrutture/sistema-pubblico-connettivita/il-nuovo-modello-interoperabilita) on the AgID website and the related list of [updates](https://www.agid.gov.it/it/argomenti/interoperabilita).

For more information on security patterns, refer to the [dedicated AgID guidelines](https://www.agid.gov.it/it/agenzia/stampa-e-comunicazione/notizie/2023/05/23/aggiornati-i-pattern-sicurezza-linee-guida-sullinteroperabilita-tecnica-pa).
