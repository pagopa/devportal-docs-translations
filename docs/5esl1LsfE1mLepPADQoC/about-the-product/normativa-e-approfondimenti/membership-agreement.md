# Membership agreement

## What the membership agreement provides

If an entity or a private party among those indicated in the [accreditation terms](further-details.md#termini-e-condizioni) completes the subscription process, it automatically gains access to all three available environments: Testing, Validation, and Production.

An entity or private party that does not belong to any of these categories may still apply. In this case, they will have access only to the Validation environment. For more information on the available environments, refer to the [dedicated section](../funzionamento-generale/ambienti-disponibili.md).

The users who initially gain access are only those designated as administrators at the time the membership agreement is signed. These users can log in to the [back office](https://selfcare.pagopa.it/) via SPID or CIE authentication and can [create additional users](../../tutorials/how-to-create-a-new-administrator.md).

For a step‑by‑step guide to the subscription process, please refer to the [dedicated tutorial](../../tutorials/guida-alladesione.md).

{% hint style="info" %}
SPID and CIE are eIDAS compliant authentication methods. For more information, see [AgID's news](https://www.agid.gov.it/it/piattaforme/eidas/cie-verso-eidas) (source in Italian).
{% endhint %}

## **Error: "Non-compliant attachment"**

If, during the process of uploading the membership agreement, you encounter the error "allegato non conforme" (non‑compliant attachment), there are two possible causes:

* The signed document was uploaded after being officially registered (“protocollato”). Registration invalidates the automatic verification of the membership agreement, and the system does not recognize it. It is necessary to register the document only after uploading it.
* The membership agreement was signed with a signature type that differs from the one required. The signature must use the CAdES standard.
