# 🖊️ What is Sign with IO?

Sign with IO is a service that allows:

- citizens to **sign documents and contracts** via the IO app in a simple, fast, and secure way;
- public administrations to **manage all signing processes** in one place.

{% hint style="success" %}
Sign with IO generates a **Qualified Electronic Signature** (QES), with a "one-shot" certificate. It has the highest probative legal value, equal to a handwritten signature, and has no regulatory exclusions.
{% endhint %}

### Benefits for citizens

<figure><img src="../.gitbook/assets/Schermata 2022-10-26 alle 12.27.11.png" alt=""><figcaption></figcaption></figure>

The feature guarantees citizens these benefits:

- **simple and secure user experience**: the protocol used ensures that the person signing is the same person from whom the public administration requested the signature, and that they did so voluntarily;
- **familiarity of the channel**: adoption is encouraged by familiarity with the IO app, which is already integrated with the signature provider;
- **multichannel service**: it can be initiated either from the IO app via a message, or from the PA's website via a button/QR code;
- **a free signature, equivalent to a handwritten one**: Qualified Electronic Signatures (QES) have the highest probative legal value.

### Benefits for public administrations

<figure><img src="../.gitbook/assets/Schermata 2022-10-25 alle 14.16.36 (1).png" alt=""><figcaption></figcaption></figure>

The feature guarantees public administrations the following benefits:

- **digitalization**: processes that require in-person signing and/or scanning of the PDF are digitized;
- **liability shift**: it guarantees a secure system that avoids any risk of repudiation;
- **simple adoption and scalability**: integration via Rest API&#x20;
